# Comparison Engine

Use this lens for every proposed design pattern before it becomes a recommendation.

## Required Comparisons

1. Current RTIH implementation.
2. Y Combinator implementation.
3. Techstars implementation.
4. Modern SaaS implementation.
5. Government or institutional innovation implementation.

## Required Conclusions

- Does it fit RTIH?
- Does it feel authentic to Visakhapatnam, Andhra Pradesh, and Indian institutional reality?
- Does it improve trust?
- Does it improve usability?
- Does it improve founder conversion?
- Does it feel globally competitive without imitation?
- Does it avoid fake metrics, generic startup visuals, and trend-led decoration?

## Recommendation Gate

A pattern can advance only when it solves a communication problem and improves clarity, credibility, usability, conversion, hierarchy, or ecosystem understanding.
